Requirements

  • Windows, MacOS, or Linux
  • Docker
  • Fully Qualified Domain Name (FQDN)
    • i.e. A hostname recognized on a public network or your institutions private network
      • Ex. myserver.mycompany.com
      • Ex. myserver.cs.university.edu
  • SSL Certificates for server
  • Optional
    • Google OAuth Credentials
    • CILogin OAuth Credentials
    • Twilio API Registration

Installation

Windows​

  1. Install Windows Subsystem for Linux WSL

  2. Install Docker within WSL subsystem

  3. Install OpenSSL and NodeJS with WSL subsystem

  4. Download SAGE3 Package

MacOS​

  1. Install Docker Desktop

  2. Install OpenSSL and NodeJS

  3. Download SAGE3 Package

  4. Extract tar

tar xvfz SAGE3-1.0-linux-arm64.tgz
  1. Change folder
cd SAGE-1.0
  1. Check to see if Docker is running
docker -v
docker ps
  1. Setup SSL
cd configurations/node/keys
# generate self-signed certificate
# optionally copy your own certificates in that folder
# add server name or IP address in the 'servers' variable in the script
# by default generate certs for HOSTNAME, localhost and 127.0.0.1
# or export HOSTNAME variable
export HOSTNAME=myserver.example.com
./genSSL-mac
  1. Setup JWT Tokens
# generate JWT token certificates (jwt_private.pem  jwt_public.pem)
./genJWT_keys.sh
# generate one (or multiple) token to give developers
# produce a 'token.json' file
./genJWT_token.sh
  1. Setup the Jupyter SSL certificates
# get back to the installation folder
cd ../../../
# Copy SSL certificates
cp configurations/node/keys/myserver-server.crt configurations/jupyter/conf/keys/certificate.crt
cp configurations/node/keys/myserver-server.key configurations/jupyter/conf/keys/private-key.key
  1. Edit .env file to set variables for your installation
vim .env
 HOST=
 FUNCX_SDK_CLIENT_ID=
 FUNCX_SDK_CLIENT_SECRET=
 ROOM_NAME=Data Science
 TOKEN= # token value copied without “” from: configurations/node/keys/token.json
  1. Pull the docker images (might takes a while)
docker compose pull
# on some system, command might be 'docker compose pull'
# Results:
# Pulling redis-server ... done
# Pulling node-server  ... done
# Pulling python       ... done
# Pulling jupyter      ... done
  1. Check that the images at downloaded
docker images
# REPOSITORY                     TAG
# jupyter/datascience-notebook   latest
# sage3/next                     amd64
# python                         3.9-bullseye
# redis/redis-stack-server       latest
